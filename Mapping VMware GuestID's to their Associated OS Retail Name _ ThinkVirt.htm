Mapping VMware GuestID's to their Associated OS Retail Name

For anyone that has spent time working with most any VMware API/SDK, used the HealthAnalyzer tool, or looked at a .vmx file, you may be familiar with the GuestId field. This value is intended to specify the guest operating system being installed on the virutal machine so the correct version of the VMware Tools package could be used.

While there is no ambiguity with "winVista64Guest," do you know which version of Windows "winNetEnterprise64Guest" refers to? Using the PowerCLI documentation as a reference, I have developed this table of GuestIds to retail names for convenience when the version of Windows or Linux a particular GuestId is referring to is not obvious, or for sharing with colleagues and clients as documentation.

These are supported GuestId values for ESX 4.0 Update 1.
